# OpenPeep Backlog

Items to consider for future development.

## Under Consideration

- **Rename project.json to .project.json** — Would auto-hide as dotfile. Requires migration logic and updating all backend/frontend references. Consider after hidden files toggle is implemented so users can still access it.
- **Image editing tools in image-viewer peep** — Add crop, rotate, resize, flip, brightness/contrast adjustments, and annotation/markup tools to the image-viewer. Would transform it from a viewer into a lightweight editor. Consider building as a separate peep (image-editor) that extends the viewer with a toolbar of editing tools and canvas-based manipulation, or extending the existing image-viewer with an edit mode toggle.
- **PeepHub user-scoped slugs & rate limiting** — Namespace peep slugs per publisher (e.g. `@username/peep-name`) to prevent slug squatting. Add rate limiting on publish endpoint (requests per minute per API key/user) and cap the number of peeps per account to prevent abuse/DoS. Consider slug reservation rules and a review queue for new publishers.
- **PeepHub similarity detection** — When publishing a peep, check for existing peeps with similar names, descriptions, or overlapping file matches to prevent near-duplicates. Could use fuzzy string matching on name/slug and compare `matches` rules. Warn the publisher or flag for admin review.
- **22 new business peeps** — See `docs/peep-priorities.md` for the full prioritized list across 5 tiers: daily operations (meeting notes, weekly reports), business tools (budget, invoice, org chart), financial (P&L, revenue projection), product development (product brief, feature list, brand guidelines, mood board, feedback log), and specialized (network diagram, hiring, headcount, compensation, roadmap).
- **Peep bundle size limits & validation** — Peeps must ship as pre-built, self-contained bundles (like browser extensions). No `package.json`, `node_modules`, or runtime dependency installation — that breaks the iframe sandbox model. PeepHub should enforce max zip size (e.g., 5MB), reject packages containing `node_modules/` or lockfiles, and validate that the entry HTML is self-contained. Authors who need external libraries should vendor/bundle them at build time or use CDN script tags. This is a feature constraint, not a gap.
- **AI-powered peep content validation** — Visual and style assessment of peep content (e.g., YouTube poll option text quality, image composition) using AI to give warnings, errors, and suggestions. Would enhance the validation panel beyond character counts and missing fields.
